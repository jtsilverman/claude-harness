# Reader model

Empty template. The COO maintains a running model of the operator as a reader here:
what they already understand (skip the recap), what terms still need an unpack, and the
altitude they prefer for a given kind of update.

It ships empty: the grader's `readerModelUpdates` accrete into this file across shipped
specs. Do not hand-author it; let the sharpen loop populate it from observed signals
(what-is-X-again questions, got-it acknowledgements, voice corrections).
